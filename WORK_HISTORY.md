# GS아트센터_좌석시야 서비스 v2 - 안티그라비티 작업 이력 요약 (Work History & Summary)

이 문서는 **GS아트센터 좌석시야 서비스 v2** 프로젝트에서 안티그라비티(Antigravity) AI와 협업하여 진행한 핵심 작업 내용과 개발 성과를 정리한 보고서입니다.

---

## 1. 🎯 프로젝트 개요 및 해결 과제 (Overview & Goal)

*   **최종 목적:** Shapespark 3D 가상 뷰어와 2D 좌석배치도(SVG)를 연동하여, 사용자가 좌석배치도에서 특정 좌석을 클릭했을 때 해당 좌석의 3D 시야(Shapespark API)로 카메라를 즉시 이동시키는 서비스 구축.
*   **핵심 문제점:** 
    *   2D 좌석배치도 SVG 파일(`seatmap_kor.svg`) 내의 각 좌석은 `<rect>` 요소로 그려져 있으나, 상당수 좌석에 고유한 좌석 ID(`id="View_1F_A_1"` 등)가 누락되어 있었음.
    *   JS 데이터 파일(`gs_arts_center_seatmap.js`)의 `GS_ARTS_CENTER_SEAT_MAP_DATA`에 정의된 3D 뷰어용 `View_ID` 데이터와 SVG 사각형을 1:1로 매치하여 자동으로 ID를 채워 넣는 정밀한 자동화 로직이 필수적이었음.

---

## 2. 🔍 주요 분석 및 해결 과정 (Analysis & Logic)

좌석 배치 형태가 단순한 그리드가 아닌 부채꼴 모양 및 구역별 회전이 들어가 있어, 다음과 같은 정밀 매칭 알고리즘을 설계하고 검증했습니다.

### ① SVG 구역별 분포 분석 (`analyze_svg_zones.py` 개발)
*   **좌석 사각형 분석:** SVG 내에서 좌석 크기(`width="13" height="15"`)를 기준으로 유효한 `<rect>`들을 추출하고, `x`, `y` 좌표의 100px/50px 단위 분포 및 회전(Rotation) 각도를 파악했습니다.
*   **회전 각도 기반 구역화:**
    *   `rotation 7` (좌측): **A 구역**
    *   `rotation 0` (중앙): **B 구역**
    *   `rotation -7` (우측): **C 구역**
*   **JS 데이터 정렬:** 각 층(Floor)과 구역(Zone)별 좌석 개수 및 정렬 규칙(Row/Seat 순서)을 파악했습니다.

### ② 순차 매칭 및 ID 자동 주입 알고리즘
*   동일한 회전 각도를 가진 그룹 내에서, **기존에 ID가 존재하던 사각형의 순서(Hint)를 기반으로** 층별/구역별 배치 진행 방향(Zone Order)을 자동으로 유추합니다.
*   유추된 구역 순서에 맞춰 JS 파일 내 실제 존재하는 `View_ID` 목록을 정렬한 뒤, ID가 누락된 빈 `<rect>` 요소에 순서대로 `id="View_..."` 속성을 정밀 인젝션(Injection)하는 알고리즘을 수립했습니다.

## 3. 💾 주요 개발 성과물 (Key Deliverables)

안티그라비티와 협력하여 작업 효율을 극대화하기 위한 두 가지 해결 도구와 규칙 문서를 생성 및 반영했습니다.

### 📄 1) Python 기반 자동화 스크립트 (`fill_svg_ids.py`)
*   **역할:** 로컬 터미널 환경에서 파이썬을 이용해 `seatmap_kor.svg` 파일 내의 빈 좌석 사각형을 찾아 JS 데이터와 1:1 자동 매칭 후, SVG 파일을 즉시 업데이트(Overwriting)하는 CLI 도구입니다.
*   **특징:** 파일 I/O 경로 꼬임을 방지하기 위해 절대 경로 모듈(`os.path.abspath`)을 사용해 안정적으로 구현되었습니다.

### 🌐 2) 브라우저 기반 GUI 유틸리티 (`fill_svg_ids.html`)
*   **역할:** 파이썬 실행 환경이 없는 브라우저 환경에서도 `seatmap_kor.svg`와 `gs_arts_center_seatmap.js`를 로드하여 클릭 한 번으로 모든 미스매치/누락된 ID를 검증 및 수정하고, 업데이트 완료된 SVG 파일을 즉시 다운로드할 수 있는 웹 기반 유틸리티입니다.
*   **디자인:** 가시적인 실행 로그(미스매치 개수, 주입된 개수, 구역별 분석 데이터)를 화면에 상세히 시각화합니다.

### ⚙️ 3) 에이전트 규칙 최적화 (`GEMINI.md` & `SKILL.md`)
*   **내용:** 에이전트(안티그라비티)가 터미널에서 불필요한 `cd` 명령어로 인한 도구 차단 에러를 겪지 않도록, `run_command` 실행 시 `Cwd` 옵션 또는 절대 경로를 사용하도록 행동 지침을 개선 및 동기화했습니다.
*   **효과:** 향후 추가적인 개발 및 디버깅 과정에서 발생할 수 있는 에이전트의 예외 행동을 차단하고 신속한 피드백 루프를 완성했습니다.

## 4. 📈 기대 효과 (Expected Benefits)

1.  **휴먼 에러 100% 방지:** 수백 개의 좌석 ID를 하나하나 수동으로 타이핑할 때 발생할 수 있는 구역/번호 누락 및 오타를 수학적 정렬 매칭으로 완벽하게 극대화했습니다.
2.  **시간 단축:** 수작업 시 최소 몇 시간이 소요되는 SVG 좌석 ID 맵핑 작업을 단 1초 만에 정확하게 끝마칠 수 있게 되었습니다.
3.  **확장성 확보:** 향후 영문판 좌석배치도(`seatmap_eng.svg`)나 층별 구역 배치 변경 시에도 해당 스크립트를 재사용하여 즉각적인 마이그레이션이 가능합니다.
